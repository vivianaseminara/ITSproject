SPARQL

In this section, we delve into the SPARQL queries to interrogate the knowledge graph. For each query, the following components are presented:

  • the query itself
  • an accompanying image showcasing the query results.
  • a concise analysis of the extracted results, when applicable.
This framework facilitates a lucid and comprehensive understanding of the executed queries and the acquired data.

1st Question:

What are the different types of architectural or landscape assets documented in the ArCo ontology?

Query:


SELECT DISTINCT ?type
WHERE {
  ?cp a arco:ArchitecturalOrLandscapeHeritage ;
        dc:type ?type .
}

Result

Analysis

The query searches for all architectural or landscape assets within the knowledge graph and extracts the unique types of these assets.

2nd Question:

What are the churches or cathedral with name and description?

Query:

SELECT ?sub ?dec
                    WHERE {
                      {
                        ?cp a arco:ArchitecturalOrLandscapeHeritage ;
                            dc:type ?o ;
                            dc:subject ?sub ;
                            dc:description ?dec .
                        FILTER(str(?o) = 'chiesa')
                      }
                      UNION
                      {
                        ?cp2 a arco:ArchitecturalOrLandscapeHeritage ;
                             dc:type ?o2 ;
                             dc:subject ?sub2 ;
                             dc:description ?dec2 .
                        FILTER(str(?o2) = 'cattedrale')
                      }
                    }

Result

Analysis

The query searches all architectural or landscape assets (represented by ?cp and ?cp2) categorized as 'church' or cathedral. For each discovered church or cathedral, the query retrieves its subject (?sub) and associated description (?dec). Note: The number of cathedrals in the graph is limited, approximately 7. Hence, the decision to focus solely on analyzing churches.

3rd Question:

What are the churches with name and description and if present the author?

Query:

SELECT ?sub ?dec ?auth
                    WHERE {
                      ?cp a arco:ArchitecturalOrLandscapeHeritage ;
                          dc:type ?o ;
                          dc:subject ?sub ;
                          dc:description ?dec .
                      OPTIONAL { ?cp a-cd:hasAuthor ?auth } .
                      FILTER(str(?o) = 'chiesa')
                    }

Result

Analysis

The query aims at identifying all architectural or landscape assets (represented by ?cp) classified as 'church'. For each church found, the query retrieves the subject (?sub), description (?dec), and, if provided, the architect (?auth).

4th Question:

How many churches are there?

Query:

SELECT (COUNT(?cp) AS ?numeroChiese)
                    WHERE {
                      ?cp a arco:ArchitecturalOrLandscapeHeritage ;
                          dc:type ?o .
                      FILTER(str(?o) = 'chiesa')
                    }

Result

Analysis

The query aims at identifying all architectural or landscape assets (represented by ?cp) categorized as 'church' and calculates the total amount of such assets. The result provides the overall number of churches within the knowledge graph.

5th Question:

What is the number of churches grouped by region?

Query:

SELECT (COUNT(?cp) AS ?numeroChiese)
                    SELECT ?regionId (COUNT(?regionId) AS ?regionCount)
                        WHERE {
                        ?cp a arco:ArchitecturalOrLandscapeHeritage ;
                            dc:type ?o ;
                        arco:regionIdentifier ?regionId .
                        FILTER(str(?o) = 'chiesa')
                        }
                        GROUP BY ?regionId
                        ORDER BY DESC(?regionCount)

Result

Analysis

The query searches for all architectural or landscape assets (represented by ?cp) classified as 'church' and linked to a region (?regionId). Subsequently, it counts the number of churches for each region and arranges the results in descending order based on the count of churches (from the highest number to the lowest). This query unveils an uneven distribution of churches across different regions. Specifically, we will further investigate the churches and architectural styles present in Sicily, which currently only has 26 identified churches under ID 19. This finding underscores certain gaps within the Knowledge Graph.

6th Question:

What are the churches built in the Baroque style in Italy?

Query:


                    PREFIX dcterms: <http://purl.org/dc/terms/>

                    SELECT ?lStile (COUNT(?stile) AS ?stileCount)
                    WHERE {
                    ?cp a arco:ArchitecturalOrLandscapeHeritage ;
                        dc:type ?o ;
                        a-cd:hasAuthorshipAttribution ?Auth ;
                        dcterms:spatial ?loc .
                    ?loc rdfs:label ?l .
                    ?Auth a-cd:hasCulturalScope ?stile .
                    ?stile rdfs:label ?lStile .
                    FILTER(str(?o) = "chiesa") 
                    FILTER(REGEX(?lStile, "barocco", "i"))
                    }
                    GROUP BY ?lStile
                    ORDER BY DESC(?stileCount)

Result

Analysis

The query seeks all architectural or landscape assets classified as "church" and featuring a cultural style containing the term "baroque". It then counts the occurrences of each baroque style and organizes the results in descending order based on frequency. Now, let's shift our attention to the architectural styles found in Italy, with a particular emphasis on the baroque style. Based on information available online, the baroque style is notably prominent in the regions of Sicily, Lazio (including Rome), and Puglia. Our intention is to corroborate this information through a query of the Knowledge Graph (KG).

7th Question:

Which are the Sicilian churches?

Query:


                    PREFIX dcterms: <http://purl.org/dc/terms/>

                    SELECT ?cp ?loc ?l ?lStile
                    WHERE {
                      ?cp a arco:ArchitecturalOrLandscapeHeritage ;
                          dc:type ?o ;
                          a-cd:hasAuthorshipAttribution ?Auth ;
                          dcterms:spatial ?loc .
                      ?loc rdfs:label ?l .
                      ?Auth a-cd:hasCulturalScope ?stile .
                    ?stile rdfs:label ?lStile .
                      FILTER(str(?o) = "chiesa") 
                      FILTER(REGEX(?l, "Sicilia", "i"))
                    }

Result

Analysis

The query retrieves all architectural or landscape assets categorized as "church" located in Sicily. For each identified church, the query provides the asset identifier (?cp), its location (?l) and the label of the architectural style (?lStile).

8th Question:

What styles are most common in Sicilian churches?

Query:


                    PREFIX dcterms: <http://purl.org/dc/terms/>

                    SELECT ?nomeStile (COUNT(?stile) AS ?stileCount)
                    WHERE {
                      ?cp a arco:ArchitecturalOrLandscapeHeritage ;
                          dc:type ?o ;
                          a-cd:hasAuthorshipAttribution ?Auth ;
                          dcterms:spatial ?loc .
                      ?loc rdfs:label ?l .
                      ?Auth a-cd:hasCulturalScope ?stile .
                    ?stile rdfs:label ?nomeStile .
                      FILTER(str(?o) = "chiesa") 
                      FILTER(REGEX(?l, "Sicilia", "i"))
                    }
                    GROUP BY ?nomeStile
                    ORDER BY DESC(?stileCount)
                    LIMIT 4

Result

Analysis

The query searches for all churches in Sicily and tallies the occurrences of each architectural style among these churches. The results are sorted based on style frequency, providing only the top 4 most common styles.

Conclusive Remarks - SPARQL Phase

Following a detailed interrogation phase of the Arco ontology, a surprising finding has emerged regarding the distribution of churches in the Sicilian territory compared to other regions. The number of churches in Sicily is significantly lower, suggesting a potential underrepresentation in the database. From an architectural perspective, an in-depth analysis has revealed that over 80 Sicilian churches are classified as Baroque, according to Wikipedia. However, only 8 of these Baroque churches are actually present in the Arco ontology database. This is particularly significant considering that the ontology includes over 350 Baroque churches in total. This discrepancy clearly indicates a notable lack of data regarding Sicilian churches, especially those of Baroque style, in the Arco database. Such a gap poses a barrier to the comprehensive and accurate understanding of Sicilian architectural heritage, highlighting the importance of a more thorough and extensive data collection and input effort. The following section will explore how the use of Large Language Models (LLMs) can be an effective tool in improving the population of the ontology, thereby contributing to a more accurate and complete representation of Sicilian churches and their historical and artistic value.

Generated Knowledge Prompting

Goal

Generated Knowledge Prompting technology consists of a method of interacting with artificial intelligence models to obtain more accurate and complete answers. This method takes advantage of the model's ability to generate and update knowledge based on the context provided. Here is how it works: Initial question without context: A question is asked of the model (e.g., ChatGPT) without providing any context. This is used to obtain a basic answer, which may be incomplete or generic. Adding context: After obtaining the initial answer, you provide the model with additional context. This context may include specific details, additional information, or clarification of the original question. Repeated question with context: You ask the same question again to the model, but this time with additional context. The model uses the new information to generate a more detailed and relevant answer. Use of enriched knowledge: The new, more complete and informative answer can be further used to query other artificial intelligence models (such as LLaMa and Gemini) or to further enrich the knowledge graph. This approach allows the quality of the answers to be progressively improved, taking full advantage of the machine learning models' ability to process and integrate new information. The goal is to understand whether the "isDedicatedTo" feature is essential to represent a church, as it was proposed during the knowledge graph enrichment phase. Recall that this feature was suggested but is not present in Arco. We begin by posing a question to ChatGPT without providing any context. ChatGPT responds with a "No." Next, we provide ChatGPT with additional information and repeat the question. This time, ChatGPT offers us additional knowledge. With the new question enriched with information, we then query LLaMa and Gemini.

Result

Analysis

In both cases, it gives us a detailed explanation of why it does not make sense to have a dedication in churches. Therefore, you may ultimately decide not to include such a feature.

Final Remarks

The initial project aimed to conduct an in-depth analysis of Italy's architectural heritage, with a particular focus on churches, to extract information on architectural styles. However, after an extensive exploration of SPARQL queries within the ArCo ontology, the results revealed a significant gap in the representation of Sicilian churches in the database. While external sources, such as Wikipedia, classify over 80 Sicilian churches as Baroque, the ArCo ontology currently documents only 8. Consequently, the main goal became to enrich the ontology with new data. For this purpose, several language models (LLMs) were used and the following observations emerged:

  1. ChatGPT: It was particularly effective in translating and converting information into the ArCo ontology, probably due to the presence of relevant data in its initial training dataset.
  2. LlaMA: While it excelled in the initial phase of information extraction, it showed significant shortcomings in the final phase of translation into the ArCo ontology.
  3. Gemini: It recorded high performance, although more in-depth answers were expected. A distinguishing feature of Gemini from other models is its ability to flag translation problems in the ArCo ontology, highlighting what it fails to translate correctly.
  4. Mistral: As an emergent model, it failed to answer the first question, probably due to its immaturity and limited training on these specific categories.
Strengths were identified for each model, and finally an example query was proposed for entering new data into the ontology.
